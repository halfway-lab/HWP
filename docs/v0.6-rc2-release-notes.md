# HWP v0.6 RC2 Release Notes

**发布版本**: v0.6 RC2  
**发布日期**: 2026-03-28  
**发布类型**: Release Candidate 2 (Runner 输出收口与本地端到端验收)

---

## 版本概述

HWP v0.6 RC2 聚焦于把 v0.6 字段从文档/测试层真正接入 runner 输出链路，并补齐在无 live provider 环境下的本地端到端验收路径。

RC1 主要完成文档、spec 草案和测试基础设施。RC2 则把这些定义推进到实际 runner 行为，使 blind spot、semantic group、continuity 三个方向能够在真实日志结构中 materialize，并被统一 verifier 稳定识别。

---

## 关键更新

### 1. Runner materialization

`runs/run_sequential.sh` 现在会在每轮输出写入日志前自动补齐以下 v0.6 字段：

- `blind_spot_signals`
- `blind_spot_score`
- `blind_spot_reason`
- `semantic_groups`
- `group_count`
- `cross_domain_contamination`
- `round_id`
- `continuity_score`
- `state_snapshot`

这些字段会被写回 `payloads[0].text` 中的内层 JSON，同时也会投影到外层日志对象，方便现有 verifier 直接读取。

### 2. Provider preflight

runner 在启动前会检查 agent 命令是否存在。

如果系统中缺失 `openclaw`，现在会直接报出明确错误，而不是在 round 1 才失败：

```text
错误：未找到 agent 命令: openclaw
请先安装/配置 OpenClaw，或通过 HWP_AGENT_BIN 指定可执行文件路径。
```

如需指定自定义 agent 路径，可设置：

```bash
export HWP_AGENT_BIN=/path/to/openclaw
```

### 3. Replay mode

为了让 RC2 在没有 live provider 的环境里仍然可以完成本地端到端验收，runner 新增 replay 模式：

```bash
export HWP_REPLAY_CHAIN_PATH=logs/chain_hwp_1772451330_20939.jsonl
HWP_ROUND_SLEEP_SEC=0 bash runs/run_sequential.sh inputs/probe.txt
```

在 replay 模式下，runner 会按轮读取已有链日志并复用同一套 enrich / repack / 写日志逻辑，因此仍然能够生成新的 RC2 格式日志并跑完整验证。

### 4. Semantic verifier 误判修正

`runs/verify_v06_semantic_groups.sh` 修正了对污染标记的判断逻辑。

之前只要日志中出现 `cross_domain_contamination` 这个字段名，就有可能被模糊 grep 误判为“跨域污染命中”。现在 verifier 只会在显式 `true` 的情况下判定失败。

## 已知边界

1. Replay 模式验证的是 runner/materialization/verification 链路，不替代 live provider 的真实生成质量评估。
2. 在当前环境中，若要运行 live provider 模式，仍需安装或配置 `openclaw`。
3. RC2 的部分字段是 runner 侧派生补全结果，用于把 v0.6 协议层定义稳定接入日志与验证链路。
