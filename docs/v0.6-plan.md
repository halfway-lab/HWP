1. **blind spot detection** - 盲区检测机制

3. **cross-round continuity** - 跨轮次连续性测试

## 3. 验证原则

1. **先文档后实现** - 所有 spec 修改必须建立在已有计划文档上
2. **先测试后集成** - test 脚本用于收敛 spec，而非无定义地乱测

4. **失败即停** - 验证失败时不允许继续扩散修改

## 6. v0.6 Increment Draft

本节为 v0.6 协议增量的文档草案，为后续 spec/hwp_turn_prompt.txt 的修改提供设计基础。

### 6.1 blind spot detection 目标

**目标**: 在 HWP 每轮生成中识别模型可能忽略的关键信息或潜在问题区域。

**检测维度**:
| 维度 | 说明 |
|------|------|
| 信息遗漏 | 用户输入中未被当前响应覆盖的关键点 |
| 逻辑断层 | 推理链条中的跳跃或缺失环节 |
| 假设冲突 | 隐含的相互矛盾的假设 |
| 边界盲区 | 极端情况或边界条件未被考虑 |

**输出字段草案**:
- `blind_spot_signals`: 检测到的盲区信号列表（数组）
- `blind_spot_score`: 盲区严重程度评分（0.0-1.0）
- `blind_spot_reason`: 判定盲区的理由说明（字符串）

### 6.2 semantic group expansion 目标

**目标**: 确保语义组在扩展过程中保持内部一致性和逻辑连贯性。

**扩展原则**:
| 原则 | 说明 |
|------|------|
| 内聚性保持 | 扩展后的语义组成员应保持高度语义相关 |
| 边界清晰 | 不同语义组之间应有明确区分 |
| 标识稳定 | 语义组标识符在扩展过程中保持一致 |

**验证要点**:
- 组内节点相似度不低于阈值
- 无跨域语义污染
- 扩展路径可追溯

### 6.3 cross-round continuity 目标

**目标**: 确保多轮生成之间的上下文连续性和状态一致性。

**连续性维度**:
| 维度 | 说明 |
|------|------|
| 状态传递 | 前一轮输出正确传递到下一轮输入 |
| 语义一致性 | 跨轮次的语义组保持逻辑连贯 |
| 无信息丢失 | 关键信息在轮次间不丢失 |
| 增量正确性 | 每轮增量内容符合预期 |

**关键字段**:
- `round_id`: 轮次标识符
- `continuity_score`: 连续性评分
- `state_snapshot`: 状态快照

### 6.4 blind spot 字段详细草案

#### blind_spot_signals
```json
{
  "blind_spot_signals": [
    {
      "type": "information_gap",
      "description": "用户询问 X，但响应未涉及 Y",
      "severity": "medium"
    }
  ]
}
```

#### blind_spot_score
- **类型**: float
- **范围**: 0.0 - 1.0
- **含义**: 
  - 0.0: 无盲区
  - 0.3: 轻微盲区
  - 0.7: 严重盲区
  - 1.0: 完全盲区

#### blind_spot_reason
- **类型**: string
- **格式**: 简短说明判定依据
- **示例**: "用户提及 A 和 B，但响应仅处理 A，未说明 B 的处理方式"
