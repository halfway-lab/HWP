# HWP Data Contracts

Last updated: 2026-03-31

## Purpose

This document defines the current machine-facing output contracts that downstream
tools can rely on in the HWP protocol repository.

It currently covers:

- benchmark overview exports under `reports/benchmarks/<timestamp>/overview.tsv`
- dashboard JSON endpoints served by `python3 hwp_server.py`

This is a practical contract document, not a planning note. If implementation
changes, this file should be updated in the same change.

## Benchmark Overview TSV

Generated by:

- `bash runs/run_benchmarks.sh`
- `python3 runs/report_benchmark_overview.py ...`

Location:

- `reports/benchmarks/<timestamp>/overview.tsv`

### Columns

| Column | Type | Meaning |
| --- | --- | --- |
| `benchmark` | string | Benchmark name derived from the input filename |
| `run_status` | string | Benchmark execution result, currently `pass` or `fail` |
| `verifier_status` | string | Aggregate verifier result across structured/blind-spot/continuity/semantic-groups |
| `duration_sec` | integer string | Whole benchmark wall-clock runtime in seconds |
| `log_count` | integer string | Number of chain log files copied into the benchmark report directory |
| `timing_chains` | integer string | Number of chain summaries matched in `logs/run.log` |
| `timing_rounds` | integer string | Sum of completed rounds across matched chain summaries |
| `round_avg_sec` | decimal string | Average round duration across matched chain summaries |
| `round_max_sec` | decimal string | Maximum observed single-round duration across matched chain summaries |
| `chain_avg_sec` | decimal string | Average chain duration across matched chain summaries |
| `chain_max_sec` | decimal string | Maximum observed chain duration across matched chain summaries |
| `failure_reason` | string | Runner-level failure classification when available |
| `report_dir` | string | Absolute path to the per-benchmark report directory |

### Semantics

- `verifier_status=pass` means all current protocol verifier dimensions passed.
- Empty timing fields mean no matching chain summary was found in `logs/run.log`.
- All numeric values are serialized as strings in the TSV output.

### Minimal Example

```tsv
benchmark	run_status	verifier_status	duration_sec	log_count	timing_chains	timing_rounds	round_avg_sec	round_max_sec	chain_avg_sec	chain_max_sec	failure_reason	report_dir
probe	pass	pass	7	1	1	8	0.56	0.62	7.08	7.08		/abs/path/to/reports/benchmarks/20260331T141118/probe
```

## Dashboard JSON API

Served by:

- `python3 hwp_server.py --host 127.0.0.1 --port 8088`

Base endpoints:

- `/api`
- `/api/dashboard`
- `/api/chain/latest`
- `/api/benchmark/latest`
- `/api/multi-provider/latest`

### `/api`

Purpose:

- lightweight API index

Response shape:

```json
{
  "endpoints": {
    "/api/dashboard": "string",
    "/api/chain/latest": "string",
    "/api/benchmark/latest": "string",
    "/api/multi-provider/latest": "string"
  },
  "query_params": {
    "/api/chain/latest": ["path=<relative-or-absolute-chain-path>"],
    "/api/benchmark/latest": ["report=<benchmark-report-dir-name>"],
    "/api/multi-provider/latest": ["report=<multi-report-dir-name>"]
  }
}
```

### `/api/chain/latest`

Optional query params:

- `path`

Response shape:

```json
{
  "chain_path": "string",
  "round_count": 8,
  "latest_round": {
    "round": 8,
    "mode": "Rg1",
    "continuity_score": 0.9,
    "blind_spot_score": 0.4,
    "collapse_detected": false,
    "recovery_applied": false
  },
  "rounds": [
    {
      "round": 1,
      "mode": "Rg0",
      "continuity_score": 1.0,
      "blind_spot_score": 0.0,
      "collapse_detected": false,
      "recovery_applied": false
    }
  ]
}
```

Notes:

- `latest_round` is the last item from `rounds`.
- Field value types depend on the underlying JSONL payloads; numbers remain numeric.

### `/api/multi-provider/latest`

Optional query params:

- `report`

Response shape:

```json
{
  "report_name": "multi_20260331T141500",
  "report_dir": "/abs/path/to/reports/benchmarks/multi_20260331T141500",
  "providers": [
    {
      "provider": "deepseek",
      "source": "overview.tsv",
      "run_pass": "3/3",
      "full_pass": "3/3",
      "timing": "3/3",
      "score_percent": "100.0",
      "avg_chain_sec": "7.20",
      "max_chain_sec": "7.69"
    }
  ]
}
```

Notes:

- When no multi-provider report exists yet, the dashboard may expose an empty object for this section.
- `source` indicates whether the provider summary was derived from `overview.tsv` or a fallback `results.tsv`.

### `/api/dashboard`

Purpose:

- combined latest snapshot used by the lightweight dashboard page

Top-level keys:

| Key | Meaning |
| --- | --- |
| `repo_root` | Absolute path to the HWP repository root |
| `latest_chain` | Absolute path to the selected/latest chain log |
| `latest_round` | Last round summary from the selected/latest chain |
| `chain_rounds` | Round-by-round protocol state list |
| `available_chains` | Recent chain log paths for dashboard selection |
| `available_benchmark_reports` | Recent benchmark report directory names |
| `available_multi_provider_reports` | Recent multi-provider report directory names |
| `selected_chain` | Requested chain path when selection is used |
| `selected_benchmark_report` | Requested benchmark report name when selection is used |
| `selected_multi_provider_report` | Requested multi-provider report name when selection is used |
| `benchmark` | Same shape as `/api/benchmark/latest` |
| `multi_provider` | Same shape as `/api/multi-provider/latest` |

## Stability Expectations

- Field names in this document should be treated as stable within the current RC2-era mainline unless explicitly changed in release notes.
- New fields may be added in a backward-compatible way.
- Downstream consumers should tolerate:
  - empty strings in TSV timing fields
  - empty objects for unavailable report sections
  - empty arrays when no recent artifacts exist
