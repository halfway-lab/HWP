# HWP v0.6 自动开发报告模板

## 使用说明

本文档作为 `reports/auto-dev-YYYYMMDD-HHMM.md` 的模板，用于统一 v0.6 阶段的自动开发报告格式。

---

## 报告字段说明

### 1. selected_task

- **ID**: 任务编号（如 V06-001）
- **Title**: 任务标题
- **Type**: 任务类型（docs/spec/test/runner）
- **Priority**: 优先级

### 2. why_selected

说明选择该任务的原因，例如：
- 是 type=docs 的最高优先级 todo 任务
- 依赖前置任务已完成
- 符合当前开发阶段目标

### 3. files_touched

列出本次修改的所有文件，格式：
- `文件路径` (操作类型: 新建/修改/删除) - 简要说明

### 4. summary

简要描述本次修改的内容，包括：
- 新增或修改了哪些内容
- 解决了什么问题
- 对 v0.6 目标的贡献

### 5. validation_commands

列出执行的所有验证命令：

```bash
# 示例
 test -f docs/v0.6-plan.md
 grep -q 'keyword' docs/v0.6-plan.md
```

### 6. validation_results

| 验证项 | 结果 | 备注 |
|--------|------|------|
| 检查项1 | PASS/FAIL | 说明 |
| 检查项2 | PASS/FAIL | 说明 |

**总体结果**: success / partial / failed / report_only

### 7. risks

列出本次修改的潜在风险：
1. 风险描述及影响
2. 缓解措施建议

### 8. next_step

建议下一步行动：
- 推荐处理的下一个任务
- 需要注意的事项
- 人工审核建议

---

## 示例报告

```markdown
# Auto-Dev Report: 2026-03-28 2230

## selected_task

- **ID**: V06-001
- **Title**: 建立 v0.6 总体开发计划文档
- **Type**: docs
- **Priority**: 1

## why_selected

V06-001 是 backlog 中 priority=1 的最高优先级任务，且 status=todo。

## files_touched

1. `docs/v0.6-plan.md` (新建) - 创建了 v0.6 总体开发计划文档

## summary

成功创建了 v0.6 总体开发计划文档，包含版本目标、阶段划分、验证原则等。

## validation_commands

```bash
test -f docs/v0.6-plan.md
grep -q 'blind spot' docs/v0.6-plan.md
grep -q 'continuity' docs/v0.6-plan.md
```

## validation_results

| 验证项 | 结果 |
|--------|------|
| 文件存在 | PASS |
| 包含关键词 | PASS |

**总体结果**: success

## risks

1. 低风险：本文档仅为规划性质

## next_step

继续处理 V06-002
```
